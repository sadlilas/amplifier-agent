Amplifier as Agent Interactive Architecture

Amplifier as Agent — Architecture

A 5-layer stdio coprocess design. The Amplifier kernel is wrapped as a callable child process — not a server. Click any layer to drill in.

💡 Click any layer above, or use the nav bar. Press Esc to return here. The dashed Shell-out chip means those callers bypass L2 and L3 and hit the L4 binary directly.
LAYER 1

Host Harnesses

External AI platforms and CLI callers that consume Amplifier Agent. Not owned by this repo.

NanoClaw

TS · containerised
  • Calls AgentProvider.query() — burst-capable; multiple push() per query.
  • Runs inside Docker containers per conversation.

Paperclip

Node · webCLI
  • Calls ServerAdapterModule.execute(ctx) — one-shot per execute.
  • Approval deferred via wakeReason: 'approval_callback'.

Shell-out callers

bypasses L2 + L3
  • bash · Codex · Claude Code · Gemini · Copilot · OpenClaw.
  • Invoke amplifier-agent run "prompt" directly — the fallback path for any host that can run a binary.
Why this drives L2: the diversity of host lifecycles (burst vs one-shot), approval mechanisms (MCP, async callback, none), and display surfaces is exactly what L2 adapters normalize away.
LAYER 2

Host-Specific Adapters

Thin TypeScript glue (~200–400 LOC each) translating host APIs into spawnAgent().

amplifier-nanoclaw-adapter

lifecycle: burst* MCP approval
  • Holds subprocess across multiple push() calls in one query.
  • Routes approval through MCP ask_user_question.
  • Needs idleHeartbeat: true to keep the container alive.

amplifier-paperclip-adapter

lifecycle: one-shot async wake
  • Fresh subprocess per execute(ctx).
  • Approval via inter-heartbeat wake-reason callback.
  • Translates AaA display events → Paperclip native format.
* 2026-05-20 correction: Both hosts now confirmed one-shot at the boundary. Wire v0 locks lifecycle: 'one-shot' only; 'burst' is reserved — v1 engine rejects with lifecycle_unsupported.
LAYER 3

Language Wrappers

SDK packages for adapter authors — same shape, two transports.

amplifier-agent-client-ts

npm · Node 18+
  • Spawns amplifier-agent subprocess; speaks JSON-RPC over NDJSON.
  • spawnAgent({lifecycle, sessionId, resume, approval, display}) → SessionHandle

amplifier-agent-client-py

in-process import
  • Uniquely embeds amplifier_agent_lib — no subprocess overhead.
  • Python hosts can skip the process boundary entirely.

Enforced invariants

  • One in-flight submit() per session — kills the V1 race.
  • L14 contract — wrapper synthesizes result/final if engine omits.
  • Env allowlist on subprocess spawn — no PATH-loss bugs.

Skew policy

strict-refuse
  • Mismatched PROTOCOL_VERSION → typed protocol_version_mismatch with remediation commands.
  • Opt-out: --allow-protocol-skew for diagnostics.
⚡ ARCHITECTURAL SEAM

Process Boundary

Wire v0 = 2026-05-aaa-v0. JSON-RPC 2.0 over NDJSON over stdio.

Client → engine (requests)

  • agent/initialize — version + capability check
  • turn/submit — user prompt, returns final result
  • agent/shutdown — graceful exit

Engine → client (notifications)

  • display/event — 9 typed event types, streaming up
  • result/final — terminal event for a turn
  • error/* — typed error events

Engine → client (request!)

  • approval/request — engine asks the host to approve a tool call
  • Mandatory timeoutMs — no infinite hangs

Out-of-band

  • D3: no turn/cancel on the wire.
  • Cancel = SIGTERM the subprocess (5s grace, then SIGKILL).
⚡ stdout is sacred — only JSON-RPC frames go there. stderr is free for logs. Lint-enforced inside amplifier_agent_lib.
LAYER 4 · PRIMARY DELIVERABLE

Engine Library + CLI Binary

Transport-agnostic library + thin CLI adapter + opinionated vendored bundle.

amplifier_agent_cli

Binary: amplifier-agent
  • Mode A: run "prompt" — argv, exit on final. ~80 LOC.
  • Mode B: run --stdio — JSON-RPC loop, exit on EOF. ~150 LOC.
  • Admin: doctor · config show · cache clear · prepare (NEW).

amplifier_agent_lib

never touches stdio
  • Library NEVER reads stdin / writes stdout — lint-enforced.
  • 3 ProtocolPoints: Approval, Display (+ streaming).
  • Python TypedDicts are the authoritative wire spec source.

Vendored bundle

D4 sealed
  • Ships in the wheel: bundle.md + 4 agents (explorer/planner/coder/tester).
  • First run: 5–30s (git clone + uv install). Warm: sub-second from XDG pickle.
  • Cache key = version + sha256(bundle.md) — edits self-invalidate.

9 display event types

  • turn/started · tool/started · tool/completed
  • assistant/text · turn/completed
  • subagent/started · subagent/completed
  • error/recoverable · provider/throttled
Streaming hook (bundle/hooks/streaming_emitter.py) bridges the kernel's event-log → ctx.display.emit(). Without it, production emits zero display events.
LAYER 5 · UNCHANGED

Amplifier Kernel

Pre-existing kernel. Consumed, never modified — forces all AaA novelty above this line.

amplifier-foundation

  • PreparedBundle — expensive once, cheap thereafter. Singleton per process.
  • load_bundle() resolves manifest, git-clones modules, uv pip install --no-sources.

amplifier-core

  • AmplifierSession — ephemeral, one per submit_turn() in v1.
  • ModuleCoordinator — drives orchestrator, providers, tools, hooks during execute().

amplifier-module-context-persistent

  • Transcript stored as JSONL on disk in XDG state.
  • On resume: replay transcript into a fresh session — no daemon needed.
EXTERNAL

LLM Provider APIs

Third-party providers consumed by the kernel. AaA has zero direct coupling.

Auto-detect (D5)

  • No config files. Env-var precedence:
  • 1. ANTHROPIC_API_KEY
  • 2. OPENAI_API_KEY
  • 3. AZURE_OPENAI_API_KEY + AZURE_OPENAI_ENDPOINT
  • 4. OLLAMA_HOST
  • Override: --provider <name>.

Errors

provider_not_configured provider_init_failed
  • Structured errors include exact env vars to set.
END-TO-END

Request Flow

How one prompt travels down the layers — and how events stream back up.

1

Host call

NanoClaw query(), Paperclip execute(), or a shell script invokes amplifier-agent run directly.

2

L2 adapter translates

Host API → universal spawnAgent({lifecycle, approval, display, ...}). Shell-out callers skip this.

3

L3 wrapper opens session

TS wrapper spawns amplifier-agent --stdio; sends agent/initialize. Python wrappers import in-process and skip the boundary.

4

⚡ Boundary crossed

JSON-RPC frames over NDJSON over stdio. turn/submit carries the prompt.

5

L4 engine boots

Engine.boot(approval, display). PreparedBundle loaded from XDG cache (5–30s cold, sub-second warm).

6

L5 kernel runs the turn

session.execute() drives orchestrator → tool modules → LLM provider.

7

Streaming flows UP

Streaming hook → ctx.display.emit() → JSON-RPC display/event → host UI.

8

Approval (if needed)

Engine sends approval/request → adapter prompts host → response flows back; tool resumes or aborts.

9

Turn completes

Engine emits result/final (or wrapper synthesizes per L14). Transcript JSONL persisted.

10

Resume next turn

Caller passes the same sessionId with resume: true → transcript replayed into a fresh session. No daemon required.

REFERENCE

Glossary

Terms and locked decisions used throughout the architecture.

AaA
Amplifier as Agent — the project name.
stdio coprocess
Child process talking to its parent via stdin/stdout. Same category as LSP servers, MCP servers, git fast-import.
Mode A
amplifier-agent run "prompt" — argv, one turn, exit. For shell-out callers.
Mode B
amplifier-agent run --stdio — JSON-RPC loop, many turns, exit on EOF. For wrapper-driven hosts.
ProtocolPoints
3 injection surfaces: Approval, Display (+streaming). Injected at Engine.boot().
SessionHandle
L3 object: submit() → AsyncIterable<DisplayEvent>, cancel(), dispose(), getEngineInfo().
Wire v0
2026-05-aaa-v0. JSON-RPC 2.0 / NDJSON / stdio. TypedDicts are source of truth.
lifecycle: one-shot
v1's only allowed value. Fresh subprocess per submit().
lifecycle: burst
Reserved enum value. v1 rejects with lifecycle_unsupported.
PreparedBundle
Foundation artifact. Expensive once (5–30s), cheap thereafter. Singleton per process.
AmplifierSession
Kernel session. Ephemeral. Rebuilt from transcript when is_resumed=True.
Vendored bundle
Manifest + 4 agent .md files shipped inside the wheel. Modules fetched on first invocation.
XDG cache
$XDG_CACHE_HOME/amplifier-agent/prepared/. Cache key includes manifest hash.
L14 contract
If engine omits result/final but reply scalar exists, wrapper synthesizes one. Tested in both TS + Py.
Streaming hook
streaming_emitter.py — translates kernel events to display events. Gap-(d) fix.
Process boundary
Seam between L3 (TS) and L4. Python wrappers bypass via in-process import.
D1–D6
Brian's locked decisions: (1) not a service, (2) seal it, (3) bake in spawn, (4) buildup bundle default, (5) env-var provider, (6) approval/display bubble up.
protocol_version_mismatch
Strict-refuse error on agent/initialize skew. Carries remediation commands.
engine_not_primed
Engine invoked before prepare. Carries exact command to fix.
Display taxonomy
9 canonical event types — turn/*, tool/*, assistant/text, subagent/*, error/recoverable, provider/throttled.
